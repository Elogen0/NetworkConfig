
### Virtual Box에서 디스크 추가후 Attach

### VM머신에서 `lsblk` 명령어로 디스크가 붙었는지 확인

```bash
master@vmmaster09:~$ lsblk
NAME                      MAJ:MIN RM  SIZE RO TYPE MOUNTPOINTS
sda                         8:0    0 50.7G  0 disk 
├─sda1                      8:1    0  538M  0 part /boot/efi
├─sda2                      8:2    0  1.8G  0 part /boot
└─sda3                      8:3    0 48.4G  0 part 
  └─ubuntu--vg-ubuntu--lv 252:0    0 48.4G  0 lvm  /
sdb                         8:16   0  100G  0 disk 
sr0                        11:0    1 1024M  0 rom
```
sdb가 생성된것을 확인했으면

### 파티션 생성
```bash
sudo fdisk /dev/sdb
```
어떻게 할지 프롬프트로 물어보는데 아래순서를 통해 파티션 생성
- Command : n
- Select : p
- Partition number : default
- First sector, LastSector : default
- Command : w
→ /dev/sdb1 생성됨

```bash
master@vmmaster09:~$ lsblk
```

### 파일 시스템 생성 (포맷)
```bash
sudo mkfs.xfs /dev/sdb1
```
왜 포맷하냐?
👉 디스크는 그냥 “빈 금속판” 상태
👉 파일을 저장하려면 파일시스템 구조가 필요함
👉 ext4 / xfs가 그 구조
이 명령어로 해당 공간에 "파일 시스템 구조"를 생성함
- Superblock
- inode 테이블
- 저널 영역
- 블록 관리 구조

### 마운트
```bash
sudo mkdir /mnt/my-storage
sudo mount /dev/sdb1 /mnt/my-storage
```
디스크가 사용할 디렉토리를 생성하고 디스크에 해당 디렉토리를 연결한다.<br>
디렉토리는 임의의 경로를 생성해도 된다. 보통은 /mnt 디렉토리 아래에 새로운 디렉토리를 생성함<br>
이제 /mnt/my-storage 디렉토리가 실제로는 /dev/sdb1 디스크 공간을 사용하게 됨

### 영구 마운트 설정
`sudo blkid` 명령어로 UUID확인하여 복사한 후 `/etc/fstab` 파일 제일 아래에 아래 추가(""없음)
```bash
UUID=xxxx  /mnt/my-storage  xfs  defaults  0  0
```

바로 재부팅 하지말고 먼저 테스트
```bash
sudo mount -a
```
fstab에 등록해놔야 부팅시 자동 마운트 된다

### 왜 이런구조가 필요한가?
/nfs 디렉토리를 네트워크로 겅유하는것
